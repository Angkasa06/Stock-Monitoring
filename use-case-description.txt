================================================================================
USE CASE DESCRIPTIONS - STOCK MONITORING SYSTEM
================================================================================

--------------------------------------------------------------------------------
1. Login
--------------------------------------------------------------------------------
Use Case Name : Authenticate User
Summary       : The admin logs into the system by providing valid credentials to access the secure dashboard.
Actor         : Admin
Main Sequence :
1. The admin requests to log into the system.
2. The system displays the login form.
3. The admin enters their username and PIN.
4. The admin requests to submit the login credentials.
5. The system validates that the credentials are correct and match an active admin account.
6. The system grants access, establishes a session, and redirects the admin to the dashboard.

--------------------------------------------------------------------------------
2. Notifikasi
--------------------------------------------------------------------------------
Use Case Name : View Stock Notifications
Summary       : The admin views automated alerts for products that have reached or dropped below their minimum stock threshold.
Actor         : Admin
Main Sequence :
1. The admin requests to view the stock notifications.
2. The system retrieves all products where the current stock is equal to or less than the minimum stock.
3. The system displays the notification page. For each alert, the system displays the product name, SKU, current stock, minimum stock, a stock bar progression, and a visual severity indicator (Habis/Empty or Tipis/Low).

--------------------------------------------------------------------------------
3. Daftar Barang
--------------------------------------------------------------------------------
Use Case Name : View Product List
Summary       : The admin views the complete list of products in the warehouse along with their current stock status.
Actor         : Admin
Main Sequence :
1. The admin requests to view the product list.
2. The system displays the complete list of products. For each product, the system displays the product icon, product name, SKU, category, price, current stock, minimum stock, and stock status.

--------------------------------------------------------------------------------
4. Barang Masuk
--------------------------------------------------------------------------------
Use Case Name : Add Incoming Stock
Summary       : The admin adds new stock quantity to an existing product in the warehouse to replenish inventory.
Actor         : Admin
Main Sequence :
1. The admin requests to add incoming stock.
2. The system displays the incoming stock form.
3. The admin selects a product, enters the incoming quantity, and optionally provides a note.
4. The admin requests to submit the incoming stock data.
5. The system displays a confirmation prompt containing the product name, the incoming quantity, and the total stock after addition.
6. The admin confirms the incoming stock process.
7. The system updates the product's stock in the database, records the activity to the system log, and displays a success message.

--------------------------------------------------------------------------------
5. Barang Keluar
--------------------------------------------------------------------------------
Use Case Name : Process Outgoing Stock
Summary       : The admin reduces the stock of a product to record items leaving the warehouse.
Actor         : Admin
Main Sequence :
1. The admin requests to process outgoing stock.
2. The system displays the outgoing stock form.
3. The admin selects a product, enters the outgoing quantity, and optionally provides a note.
4. The admin requests to submit the outgoing stock data.
5. The system validates that the current stock is sufficient for the requested outgoing quantity.
6. The system displays a confirmation prompt containing the product name, the outgoing quantity, and the remaining stock after deduction.
7. The admin confirms the outgoing stock process.
8. The system updates the product's stock in the database, records the activity to the system log, and displays a success message.

--------------------------------------------------------------------------------
6. Daftarkan Produk Baru
--------------------------------------------------------------------------------
Use Case Name : Register New Product
Summary       : The admin registers a brand-new product that is not yet in the system into the warehouse inventory.
Actor         : Admin
Main Sequence :
1. The admin requests to register a new product.
2. The system displays the new product registration form.
3. The admin enters the product name, SKU, category, unit type, price, initial stock, and minimum stock.
4. The admin requests to submit the registered product data.
5. The system validates that all required data (e.g., product name and valid price) has been filled.
6. The system saves the new product into the database and displays a success message.

--------------------------------------------------------------------------------
7. Cek Laporan
--------------------------------------------------------------------------------
Use Case Name : View Outflow Report
Summary       : The admin views an audit report chart of the most depleted (outgoing) items based on a custom date range.
Actor         : Admin
Main Sequence :
1. The admin requests to view the outflow report.
2. The system displays the report page with empty start date and end date fields, and prompts the user to select the dates.
3. The admin selects the start date and the end date using the calendar picker, then applies the filter.
4. The system validates that the date range is valid (Start Date is not exceeding End Date).
5. The system displays a bar chart reporting the most depleted items. For the chart, the system displays the names of the products and their total outgoing quantities within the specified date range.

--------------------------------------------------------------------------------
8. Log Sistem
--------------------------------------------------------------------------------
Use Case Name : View System Log
Summary       : The admin views the immutable, automatically generated audit trail of all warehouse stock changes.
Actor         : Admin
Main Sequence :
1. The admin requests to view the system log.
2. The system displays the system log page.
3. The admin optionally selects a log type filter (Barang Masuk, Barang Keluar, or Produk Baru).
4. The system retrieves the relevant activity logs based on the filter.
5. The system displays a paginated list of logs. For each log entry, the system displays the timestamp, log type, product name, SKU, quantity change, stock before, stock after, note, and the admin who performed the action.
